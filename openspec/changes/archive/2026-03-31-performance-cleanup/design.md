## Context

The project currently loads Inter and Manrope from Google Fonts CDN in `index.html`. This adds an external DNS lookup and potentially render-blocking request on every cold load. The repo also tracks TypeScript build artifacts (`.tsbuildinfo` files) and has duplicate/ghost configuration files (`vite.config.js`, `vite.config.d.ts`, root-level `requirements.txt`) that create confusion about source-of-truth.

No new features are introduced — this is purely build and load performance cleanup.

## Goals / Non-Goals

**Goals:**
- Eliminate Google Fonts CDN dependency; serve fonts from the Vite bundle
- Stop tracking generated build artifacts in git
- Remove duplicate/ghost config files so there is exactly one authoritative file per concern
- Reduce confusion for future contributors about which config is "real"

**Non-Goals:**
- Changing which fonts are used (Inter + Manrope stay; `design-refresh` handles font selection)
- Subsetting or advanced font optimization (WOFF2 is already well-compressed)
- Changing the Tailwind or PostCSS config content
- Any runtime behavior changes

## Decisions

**Font hosting: `@fontsource` packages over manual WOFF2 downloads**

`@fontsource/inter` and `@fontsource/manrope` are maintained npm packages that provide pre-built WOFF2 files. Importing them in CSS means Vite bundles the font files automatically, generates correct content-hash filenames for cache invalidation, and keeps fonts co-located with the app rather than in a `public/fonts/` folder that needs manual management.

Alternative considered: download WOFF2 files manually into `public/fonts/` and write `@font-face` declarations by hand. Rejected because it requires manual updates when font versions change and adds files that need to be committed.

**Fontsource import location: `src/styles/index.css` over `src/main.tsx`**

Fontsource packages export both a CSS file and a JS import. Importing the CSS file in `index.css` (which is already the global stylesheet entry point) keeps font loading in the CSS layer where it belongs, rather than bloating the JS bundle entry.

**`*.tsbuildinfo` in `.gitignore` at repo root**

These files are generated by `tsc --incremental` on every build. They are machine-local and should never be committed. Adding them to the root `.gitignore` covers both `tsconfig.app.tsbuildinfo` and `tsconfig.node.tsbuildinfo`.

**Delete `vite.config.js` and `vite.config.d.ts`**

These files appear to be artifacts of a previous build or editor tooling. `vite.config.ts` is the TypeScript source of truth. Vite does not need both `.ts` and `.js` versions — it prefers `.ts` when present. Deleting the `.js` and `.d.ts` eliminates the ambiguity.

**Delete root `requirements.txt`**

`scripts/requirements.txt` is the canonical Python dependency file. The root-level duplicate is identical and serves no purpose (no tooling or CI references it separately based on the current codebase). Deleting it prevents divergence.

## Risks / Trade-offs

- **Bundle size increase** → Fontsource adds WOFF2 font files to the Vite output. Mitigated: these files are served with long-lived cache headers and only loaded once. The total size is comparable to what the CDN served.
- **Font subsetting** → `@fontsource` packages serve full character sets by default. If only Latin is needed, the `-latin` subset packages (`@fontsource/inter/latin.css`) can reduce file size further. Not blocking for this change.
- **First deploy font cache miss** → On initial deploy after this change, users will download fonts from the app domain instead of the CDN. After that, they are cached normally.

## Migration Plan

1. Install `@fontsource/inter` and `@fontsource/manrope`
2. Add CSS imports in `src/styles/index.css`
3. Remove `<link>` tags and `preconnect` from `index.html`
4. Verify fonts render correctly (`npm run dev`, visual check)
5. Add `*.tsbuildinfo` to `.gitignore`
6. Delete `vite.config.js`, `vite.config.d.ts`, root `requirements.txt`
7. Run `npm run build` and confirm no errors
8. Commit

Rollback: revert the `index.html` and `index.css` changes; fonts fall back to CDN immediately.
